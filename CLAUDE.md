# CLAUDE.md — Technical Reference for AI Sessions

## Data Pipeline Overview

All player data flows through a pipeline from raw sources → generated JS files → browser.

### Source Files (human-managed)
- `js/data/seed.js` — Base player pool. ~543 players from Mr. CheatSheet 2026 export + Steamer fill-ins. Fields: id, n, t, pos, inj, age, csValA, csValS, IP/W/K/ERA/WHIP (pitchers) or PA/OBP/HR/XBH/RP/SB (hitters).
- `data/manual/*.csv` — Hand-curated rankings per position group. Add a column to any CSV and it auto-appears as a sortable toggle in the auction board. Columns: Name + any data columns. Current files: sp_rankings.csv (PL_Rank, PL_Tier), rp_rankings.csv (CM_Role, PL_CloserRank), hitters.csv (Name only).

### Generated JS Files (do not edit)
- `js/data/player_ids.js` — ESPN, FanGraphs, MLB IDs per player. Generated by `fetch_rankings.py`. Run once per year or when adding players to seed. Uses Chadwick Bureau (7 shards) + ESPN Fantasy API.
- `js/data/rankings.js` — ECR, ESPN auction$, ADP, closerStatus. Generated by `fetch_rankings.py --ranks`. Run weekly during draft prep.
- `js/data/manual_rankings.js` — Merged from data/manual/*.csv. Generated by `bake_manual.py`. Auto-run after fetch_rankings.py.
- `js/data/steamer_extras.js` — FanGraphs Steamer estimates for players not in seed. Generated by `generate_extras.py`. Unofficial only.

### One-Command Update
```bash
python3 scripts/fetch_rankings.py   # fetches ECR + ESPN + CloserMonkey → runs bake_manual.py automatically
python3 scripts/generate_extras.py  # refreshes unofficial Steamer estimates
```
Full rebuild (new season or after adding players to seed):
```bash
python3 scripts/fetch_rankings.py --ids  # slow: also rebuilds player_ids.js via Chadwick
```

## Data Sources

### FantasyPros ECR
- URL: `https://partners.fantasypros.com/api/v1/consensus-rankings.php?...`
- Auth: none required
- Fields: ecr (float rank), adp (float)
- Matching: normalize(name) → seed player

### ESPN Fantasy
- URL: `https://lm-api-reads.fantasy.espn.com/apis/v3/games/flb/seasons/2026/players?view=kona_player_info`
- Auth: none for public data
- Fields: espnAuction ($), espnId (for player_ids.js)
- Matching: espnId from player_ids.js

### CloserMonkey
- URL: `https://www.closermonkey.com/`
- Table structure: 2 teams per row, 10 cells total (offset 0 and 5 for each team)
- Cell layout per team: [0] team_name, [1] status_role, [2] pitcher_name, [3] pitcher_name2, [4] pitcher_name3
  - Wait, re-check this - the actual structure has CLOSER, 1ST, 2ND rows per team block
- Status encoding in rankings.js: `STATUS:TEAM` or `STATUS:TEAM:*` where * = closer-by-committee
- Status values: CLOSER, 1ST (1st in line), 2ND (2nd in line)
- Matching: last-name + team disambiguation, restricted to pure RPs

### FanGraphs Steamer Projections
- URL: `https://www.fangraphs.com/api/projections?type=steamer&stats=pit&pos=p&team=0&players=0&lg=all`
- Fetch: requires `--compressed` (gzipped) and Referer header
- Fields: PlayerName, Team, IP, W, SO (K), SV, HLD, ERA, WHIP, FPTS (auction pts), SPTS (season pts), playerid (FG id)
- Matching: name_key(PlayerName) → seed player
- csValA calibration: median(seed_csValA / FG_FPTS) for matched SPs/RPs
- Cache: /tmp/fg_steamer.json (re-fetch if missing)
- SP ratio_a ≈ 0.02180, SP ratio_s ≈ 0.01568 (recalculated each run from seed overlap)
- RP ratio_a ≈ 0.00972, RP ratio_s ≈ 0.01025 (recalculated each run from seed overlap)
- Outlier exclusion: FPTS < 500 (SPs) or < 100 (RPs), ratios outside 0.001–0.05

### PitcherList
- SP URL: `https://pitcherlist.com/top-100-starting-pitchers-for-2026-fantasy-baseball-3-13-update/`
- HTML structure: `<td class="rank">N</td><td class="name"><a>Name</a>` with `<span class="tier">TN</span>` on new-tier rows
- Stored in: data/manual/sp_rankings.csv (PL_Rank, PL_Tier columns)
- Tier starts: new-tier rows have class="new-tier" on the <tr>

### Chadwick Bureau (ID crosswalk)
- URL: `https://raw.githubusercontent.com/chadwickbureau/register/master/data/people-{0..6}.csv`
- Fields used: key_mlbam, key_fangraphs, name_first, name_last, name_suffix
- Output: player_ids.js with mlbId, fgId per player

## Data Model

### Player object in AppState.players
Core (from seed.js): id, n, t, pos[], inj, age, csValA, csValS
Pitching stats: IP, W, K, ERA, WHIP, SVH
Hitting stats: PA, OBP, HR, XBH, RP, SB
Enriched by player_ids.js: espnId, fgId, mlbId
Enriched by rankings.js: ecr, espnAuction, adp, closerStatus, closerRank
Enriched by manual_rankings.js: any CSV columns (PL_Rank, PL_Tier, CM_Role, etc.)
Draft state (in AppState.drafted): cost, team
Unofficial flag: unofficial=true for players from steamer_extras.js

### ID format
`normalize(name)-team.toLowerCase()` where normalize = lowercase + strip non-a-z0-9.
Examples: `tarikskubal-det`, `yuseikikuchi-laa`, `hoganharris-ath`

### Team normalization
Raw FG teams → seed teams: KCR→KC, SFG→SF, TBR→TB, CHW→CWS, WSN→WSH
Athletics use ATH (Sacramento move; FG uses ATH).

### CM_Role encoding
Format: `STATUS:TEAM` or `STATUS:TEAM:*` (asterisk = committee)
Status values: `CLOSER`, `1ST`, `2ND`
Example: `CLOSER:NYY` or `1ST:BOS:*`

### Share URL format
Compact: `seedIndex.cost.teamNum` per pick joined by `_`
Team numbers: me=0, t1-t10=1-10
Old format (base64) auto-detected and parsed for backwards compat

## Column System
Static columns: csValS (SEASON), csArb (ARB), ecr (ECR), espnAuction (ESPN$), projections (PROJ)
Manual columns: auto-discovered from MANUAL_RANKINGS keys, stored in AppState.manualCols
Hidden columns: AppState.ui.hiddenCols[] — toggled via column toggle bar above table
CM_Role column: renders as colored closer badge (CLOSER=green, 1ST=yellow, 2ND=blue)

## Adding New Players to seed.js
1. Find the player in /tmp/fg_steamer.json (re-fetch if missing)
2. Calculate csValA = FPTS * ratio_sp_a (or ratio_rp_a for RPs), csValS = SPTS * ratio_sp_s
3. Add the player object to SEED_PLAYERS array with correct id format
4. Sort all players by csValA descending
5. Run `python3 scripts/fetch_rankings.py` to get IDs and rankings
6. Run `python3 scripts/bake_manual.py` to merge manual CSV data
7. For official Steamer values, players in seed.js take precedence over steamer_extras.js

## steamer_extras.js
- Generated by `scripts/generate_extras.py`
- Criteria: SPs with 80+ projected IP, RPs with 5+ SV or 10+ HLD
- Players in seed.js are excluded (by name_key match)
- All extras have `unofficial: true` flag
- Rendered with 55% opacity and "(est)" badge in the UI
- Loaded in index.html between seed.js and player_ids.js
- Merged into AppState.players in components.js after seed data
